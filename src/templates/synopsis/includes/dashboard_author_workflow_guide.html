How this works for authors

The portal is meant to bring the main synopsis work into one place, so you do not have to jump between as many separate files, tools, and email threads.

Pilot guidance
This pilot is here to test how well the portal supports the team’s current way of working. Some parts already feel close to the existing process, while other parts still need feedback and improvement.

Think of each synopsis as one main workspace with several connected sections.

  • Dashboard = the home page showing the synopses you can open.
  • Project hub = the main page for one synopsis.
  • Protocol / Action list = the main shared documents authors work on.
  • Reference Database = the shared library of references across the portal.
  • Project references = the place where references for one synopsis are screened.
  • Summary board = the place where included studies are assigned and written up.
  • Evidence workspace = the place where summaries are organised into the structure of the synopsis.
  • Advisory board = the area for inviting reviewers and tracking their responses.

  1. Open a synopsis from the dashboard.
  2. Use the project hub to move into the area you need.
  3. Draft and revise the protocol and action list.
  4. Add references either from the shared reference database or by importing a search batch into that synopsis.
  5. Screen those references inside the project.
  6. Write summaries only for the references marked as included.
  7. Place those summaries into the evidence structure so they feed into the synopsis itself.
  8. Use the advisory board area to contact external reviewers and track their responses.

The portal is not trying to replace the team’s way of working with something completely different. It is trying to bring the main parts of that work together in one connected place.

  • A synopsis/project is roughly the same as the working space the team already keeps for one synopsis.
  • The global reference database is the closest thing to a shared EndNote-style library, because references can be reused across synopses.
  • Project references are closer to the screened working list for one synopsis.
  • The summary board is the portal version of assigning papers to authors and writing up what each study found.
  • The evidence workspace is where those written summaries are organised into the final synopsis structure.
  • The collaborative editing area for protocol and action list is there to reduce the need to pass document versions around separately.

  • The portal does not currently work as a full one-for-one EndNote replacement.
  • The shared reference library stores the main reference records, but screening, folders, notes, and summaries happen inside each synopsis.
  • Some parts of the portal are already quite solid, while some parts are still being refined.
  • The reference library is one of the areas still developing, so if it feels unclear or different from the current workflow, that is useful feedback.
  • There are now page-level How this works buttons in key places such as protocol, action list, the reference library, project references, and summary pages.

The overall aim is to make the synopsis workflow easier to follow and less fragmented.

  • Fewer disconnected files and manual handovers.
  • A clearer path from importing references, to screening, to writing summaries, to building the synopsis.
  • Better visibility on who is doing what.
  • Better reuse of references and evidence across projects over time.
  • Smoother collaboration with advisory board members and document contributors.

Please try to use the portal as if you were really working in it, rather than only clicking around briefly.

  • Try the workflows end to end where possible.
  • Notice where the app matches the current team workflow well, and where it does not.
  • Flag anything confusing, slow, misleading, or missing.
  • Include examples where a page or action did not behave as you expected.
  • If something feels unclear, the issue may be the wording, the page layout, or the workflow itself. All of that is useful feedback.
