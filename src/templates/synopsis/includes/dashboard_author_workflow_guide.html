How this works for authors

The portal is meant to support the main Conservation Evidence synopsis workflow in one place, so authors do not have to rely as much on separate files, spreadsheets, reference managers, and email chains.

Pilot guidance
This pilot is here to test how well the portal supports the real CE synopsis workflow from start to finish, from setting up a synopsis and gathering references through to writing summaries, building the evidence structure, and collaborating on documents and review.

Workflow step
  • Write and review the protocol
  • Develop the intervention list
  • Plan the synopsis structure
  • Search and gather the literature
  • Write summary paragraphs
  • Write key messages and background text
  • Review with the advisory board
  • Prepare the final synopsis and website content
Portal section
  • Protocol page
  • Action list and evidence workspace
  • Synopsis structure / evidence workspace
  • Reference Database and Project references
  • Summary board and summary detail pages
  • Evidence workspace and document pages
  • Advisory board pages
  • Document and evidence areas across the project

  • Dashboard = the home page showing all synopses you can open.
  • Project hub = the main page for one synopsis, with links to all the important sections.
  • Protocol = where the synopsis protocol is uploaded, revised, discussed, and shared.
  • Action list = where the intervention list and related working document are managed.
  • Reference Database = the shared library of references across the whole portal.
  • Project references = the screening area for references inside one synopsis.
  • Summary board = where included studies are assigned and written up as summary records.
  • Evidence workspace = where summaries are placed into chapters, subheadings, interventions, and key messages.
  • Advisory board = where reviewers are invited and their responses are tracked.

The portal is meant to support the existing CE process, not replace it with something unfamiliar.

  • A synopsis in the portal is roughly the same as the shared folder or main working space the team already keeps for one synopsis.
  • The shared reference library is the closest thing in the portal to a shared EndNote-style library of references.
  • Project references are closer to the screened working list for one synopsis.
  • The project references area and summary board together are the closest thing to the synopsis skeleton or tracking matrix that might otherwise sit in Excel, Access, or a shared spreadsheet.
  • The protocol and action list pages are the portal versions of the working documents that would otherwise be passed around as separate files.
  • The evidence workspace is where summaries are organised into the intervention structure of the synopsis and prepared for the website-facing workflow.
  • The portal should reduce some of the need to move between Word documents, spreadsheets, reference software, and email threads, but it does not yet replace every part of that older workflow exactly.

  • The portal does not currently work as a full one-for-one EndNote replacement.
  • The shared reference library stores the main reference records, but screening, folders, notes, and summaries happen inside each synopsis.
  • Searching and screening are still active author tasks. The system helps organise references, but authors still decide what is relevant and what should be included.
  • The summary board is where included studies are turned into usable summary records for the synopsis.
  • The evidence workspace is where those summaries are linked into chapters, interventions, and key messages.
  • The final synopsis process still includes steps outside the portal, such as CE feedback, advisory board review, and final checking before publication.
  • There are page-level How this works buttons in key areas such as protocol, action list, the reference library, project references, and summary pages for more detailed local guidance.

The overall aim is to make the synopsis process easier to follow and less fragmented.

  • Fewer disconnected files and manual handovers.
  • A clearer path from references, to screening, to summaries, to building the synopsis itself.
  • Better visibility on who is doing what.
  • Better reuse of references and evidence across projects over time.
  • Smoother collaboration with advisory board members and document contributors.

Please try to use the portal as if you were really working in it, rather than only clicking around briefly.

  • Try the workflows end to end where possible.
  • Notice where the app matches the current team workflow well, and where it does not.
  • Flag anything confusing, slow, misleading, or missing.
  • Include examples where a page or action did not behave as you expected.
  • If something feels unclear, the issue may be the wording, the page layout, or the workflow itself. All of that is useful feedback.
