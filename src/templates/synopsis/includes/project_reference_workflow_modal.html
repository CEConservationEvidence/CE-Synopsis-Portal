How project references work
Per-synopsis batches, screening, and when summaries appear.
The basic workflow
  1. Add references to this synopsis by linking from the global library or importing a RIS/plain-text file.
  2. Each import or link action creates a project batch that can be screened.
  3. Screen references as included, excluded, or pending.
  4. Included references automatically appear on the summary board.
  5. Create and assign summary tabs, then connect summarised evidence into the synopsis structure.
Two ways to add references
Link from library

Best when a reference already exists in the shared database or may be reused across projects. Linking creates a synopsis copy for screening without removing it from the library.

Import directly into this synopsis

Best for a new search result set that belongs mainly to this synopsis. The project gets a new batch, and the system still creates or reuses a matching library record in the background.

What is project-specific

Once a reference is in a synopsis, the working record becomes project-specific.

  • Screening decisions are per synopsis.
  • Reference folders are per synopsis.
  • Screening notes and comments are per synopsis.
  • Summary assignments and evidence structure links are per synopsis.
What happens after screening

Only references marked as Included move into the summary workflow.

  • The summary board groups all included references for that synopsis.
  • Each included reference gets at least one summary tab.
  • You can create additional tabs when the same paper needs more than one summary entry.
  • Summaries can be assigned to authors, moved through status columns, and flagged as needing help.
How this behaves compared with the current EndNote process

It does not mirror the current team workflow one-for-one. A better way to think about it is:

  • Global library = shared source of references
  • Project references = synopsis working set for screening
  • Summary board = project-specific evidence extraction and writing
Current state of the feature

The global reference database is already useful for reuse and deduplication, but it is still under active development. If something feels unclear or behaves differently from your current EndNote-based workflow, that is useful pilot feedback.
