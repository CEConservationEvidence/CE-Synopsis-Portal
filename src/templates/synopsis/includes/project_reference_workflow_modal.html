How project references work
How references are added to one synopsis, screened, and turned into summaries.
The basic workflow
  1. Add references to this synopsis by linking from the global library or importing a RIS/plain-text file.
  2. Each import or link creates a batch that can be screened.
  3. Screen references as included, excluded, or pending.
  4. Included references automatically appear on the summary board.
  5. Create and assign summary tabs, then connect those summaries into the synopsis structure.
Two ways to add references
Link from library

Best when a reference already exists in the shared library or may be useful in more than one synopsis. Linking creates a synopsis copy for screening without removing it from the shared library.

Import directly into this synopsis

Best for a new search result set that mainly belongs to this synopsis. The synopsis gets a new batch, and the system still creates or reuses a matching record in the shared library in the background.

What is project-specific

Once a reference is in a synopsis, the working version of that reference belongs to that synopsis.

  • Screening decisions are per synopsis.
  • Reference folders are per synopsis.
  • Screening notes and comments are per synopsis.
  • Summary assignments and evidence structure links are per synopsis.
What happens after screening

Only references marked as Included move into the summary workflow.

  • The summary board groups all included references for that synopsis.
  • Each included reference gets at least one summary tab.
  • You can create additional tabs when the same paper needs more than one summary.
  • Summaries can be assigned to authors, moved through status columns, and flagged as needing help.
How this behaves compared with the current EndNote process

It does not copy the current team workflow exactly. A simple way to think about it is:

  • Shared library = the main source of references
  • Project references = the working list for one synopsis
  • Summary board = the place where included studies are written up and assigned
Current state of the feature

The shared reference library is already useful for reuse and avoiding repeated imports, but it is still under active development. If something feels unclear or behaves differently from your current EndNote-based workflow, that is useful pilot feedback.
