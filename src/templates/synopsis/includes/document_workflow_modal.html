How this {{ document_label|lower }} workflow works

This page manages the current working {{ document_label|lower }} for the project.

  1. Upload the first {{ document_label|lower }} file here to create the working document for the project.
  2. After that, replacing the file creates a new revision. Use the version label and change reason so the team can see what changed and why. The change reason is optional for the first upload, but expected for later revisions or stage changes.
  3. When a file is active, the page shows one current revision. That is the copy authors should treat as the live working version.
  4. If you want to edit offline, download the current revision first so you are working from the latest file.
  5. Collaborative editing is optional. Authors can either upload revised files manually or use the collaborative editor to work on the same current revision live.
  6. Keep the {{ document_label|lower }} in Draft while it is still being edited. Set it to Final only when the authors agree it is ready. If more changes are needed later, return it to Draft before replacing the file.
  7. The revision history lower on the page keeps older uploads. From there you can download an earlier version, restore it as the current revision, or mark a specific revision as final.
  8. The feedback window section is for advisory board review of the {{ document_label|lower }}. It controls the deadline, reminder timing, and whether the review links are open or closed. Closing that window also disables collaborative editing for that document.
In practice: keep authors aligned around the current revision shown at the top of the page, and avoid editing an older local copy unless you plan to upload it back as a new revision.
